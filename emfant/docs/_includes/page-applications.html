Simulation Programs

Spontaneous Movement

Generate fetal spontaneous movements using spinal cord neural circuits. No explicit motor commands are given - movement emerges from body-environment dynamics.

11_infant_spinal_cord_circuit.py

Motion Capture Replay

Reproduce actual infant movements from motion capture data. Useful for validating simulation against real observations.

21_infant_mocap_joint_angle.py

Joint Position Control

Direct control of joint angles using position actuators. Simplest control mode for testing postures and movements.

01_infant_joint.py

Muscle Activation

Control movements through muscle activation signals. More biologically realistic than joint position control.

02_infant_muscle.py

Tactile Simulation

Record tactile experiences during movement. Essential for studying sensorimotor development and body representation.

03_infant_joint_with_tactile.py

Servomotor Control

Motor control with servomotor-like feedback. Useful for precise trajectory following.

Data Logging

HDF5 Format

Simulation data is saved in HDF5 format with the following structure:

import h5py

hf = h5py.File("dataset.h5", "r")
print(hf.attrs["dt"])              # Timestep (e.g., 0.001)
print(hf.keys())                   # Available data keys
print(hf["mj_joint_pos"])          # Joint positions (n_joint x 3)
print(hf["mj_joint_pos"].attrs["name"])  # Joint names

Available data includes:

  • mj_joint_pos, mj_joint_vel - Joint states
  • bvp_x, alpha, gamma - Neural activities
  • muscle_activation, ia, ib - Muscle/sensor data
  • tactile_activation - Tactile sensor readings

Reinforcement Learning

EMFANT environments are compatible with Gymnasium for RL experiments:

import gymnasium as gym
import mujoco_infant.envs

# Create environment
env = gym.make(
    id="ISICallbackJntPosActuatorEnv-v0",
    model_path="isi://250501_11_jointpos_uterus.xml",
    render_mode="human",
)

# Standard Gymnasium interface
observation, info = env.reset(seed=42)

for _ in range(1000):
    action = env.action_space.sample()  # or your policy
    obs, reward, terminated, truncated, info = env.step(action)
    
    if terminated or truncated:
        observation, info = env.reset()

For custom reward functions, see program/simulation_negaeri/ for examples of defining custom environments.
