Requirements

Python

3.9+ (Anaconda recommended)

MuJoCo

3.3.0+ (ISI modified version)

Gymnasium

1.0.0a1+

Git LFS

For large model files

System Dependencies (Ubuntu)

sudo apt update
sudo apt install git git-lfs build-essential cmake libglfw3-dev
sudo apt install libgl1-mesa-dev libxinerama-dev libxcursor-dev libxrandr-dev libxi-dev ninja-build

Installation Steps

1

Clone Repositories

# Clone MuJoCo (ISI modified version)
git clone http://gitlab/baby_simulator/mujoco.git

# Clone mujoco_infant
git clone http://gitlab/baby_simulator/mujoco_infant.git
cd mujoco_infant
git lfs install
git lfs pull
cd ..

# Clone mujoco_infant_application
git clone http://gitlab/baby_simulator/mujoco_infant_application.git
cd mujoco_infant_application
git lfs install
git lfs pull
cd ..

# (Optional) Clone osim2mujoco
git clone http://gitlab/baby_simulator/osim2mujoco.git
2

Create Conda Environment

cd mujoco_infant_application
conda env create -f ./environment_general.yml
conda activate mujoco_infant
Note: Pre-defined environment files are available for specific Ubuntu versions: environment_20_04.yml, environment_24_04.yml
3

Set Environment Variables

# Example paths - adjust for your system
export MUJOCO_PATH=/home/user/.mujoco/mujoco-3.3.0
export MUJOCO_PLUGIN_PATH=/home/user/.mujoco/mujoco-3.3.0/bin/mujoco_plugin

mkdir -p $MUJOCO_PATH
4

Build MuJoCo (ISI Modified)

cd mujoco
git checkout develop
mkdir -p build
cd build
cmake .. -DCMAKE_INSTALL_PREFIX=$MUJOCO_PATH -DCMAKE_BUILD_TYPE=Release
cmake --build . -j16
cmake --install .
cd ..

Test the build:

./build/bin/simulate ../mujoco_infant/mujoco_infant/envs/assets/240703_01_isi_infant_uterus_bvp.xml
5

Install MuJoCo Python Bindings

cd python
bash make_sdist.sh
cd dist
pip install mujoco-3.3.0.tar.gz
6

Install mujoco_infant

cd ../../..
cd mujoco_infant
pip install -e .

Running Examples

Desktop Environment

cd mujoco_infant_application
PYTHONPATH=. python program/simulation/01_infant_joint.py

Server with X Server

# Check if X server is running
ps -ef | grep -i xorg

# Run with EGL (headless)
MUJOCO_GL=egl PYTHONPATH=. python program/simulation/01_infant_joint.py

Server without X Server

# Requires xvfb: sudo apt install xvfb
PYTHONPATH=. xvfb-run -s "-screen 0 1000x1000x24" python program/simulation/01_infant_joint.py
Ubuntu 24.04: If you encounter GLIBC errors, add import mujoco at the top of your scripts before other imports.

Quick Start Code

import gymnasium as gym
import mujoco_infant.envs  # Register environments

# Create environment
env = gym.make(
    id="ISICallbackJntPosActuatorEnv-v0",
    model_path="isi://240703_03_isi_infant_uterus_mocap.xml",
    render_mode="human",
)

# Reset environment
observation, info = env.reset(seed=42, options={})

# Run simulation loop
while True:
    action = env.action_space.sample()
    observation, reward, terminated, truncated, info = env.step(action)
    
    if terminated or truncated:
        observation, info = env.reset()

Simulation Speed

The default timestep is 0.001s (1ms). You can increase the timestep for faster simulation:

Timestep Speed Stability
0.001s (1ms) Slowest Most stable
0.002s, 0.005s Medium Good compromise
0.010s (10ms) Fastest May be unstable with muscles/spinal circuits

To change timestep, modify <option timestep="0.001"> in the model XML file.

Updating MuJoCo

When a new version of MuJoCo is released and the ISI modified version is updated:

# 1. Remove old packages
conda activate mujoco_infant
pip uninstall mujoco_infant mujoco gymnasium

# 2. Install updated gymnasium
pip install --pre gymnasium

# 3. Pull and rebuild MuJoCo
cd mujoco
rm -rf ./build
git checkout develop
git pull origin develop
# Then follow build steps 4-5 above

# 4. Pull and reinstall mujoco_infant
cd ../mujoco_infant
git pull origin master
pip install -e .

# 5. Pull mujoco_infant_application
cd ../mujoco_infant_application
git pull origin master

Troubleshooting

GLIBC Error on Ubuntu 24.04: Add import mujoco as the first import in your script.
Rendering Issues: Try different rendering backends: MUJOCO_GL=egl, MUJOCO_GL=osmesa, or MUJOCO_GL=glfw
Git LFS Issues: Ensure Git LFS is installed (git lfs install) and run git lfs pull in each repository.
